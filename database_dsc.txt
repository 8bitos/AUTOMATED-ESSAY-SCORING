Table Descriptions (Automatic Essay Grading Focus)

users:
The users table stores all accounts that interact with the essay grading system. Each user has a UUID primary key, login credentials, and a role (student, teacher, or superadmin) that controls access. This table is the main identity reference for submissions and reviews.

materials:
The materials table stores learning materials that provide context for essay questions. Each material has a UUID primary key and a title, and it acts as the content source for questions and modules.

modules:
The modules table stores uploaded module files that belong to a material. Each module references one material and includes a name and file URL. This enables content to be organized in smaller units within a material.

essay_questions:
The essay_questions table stores the essay prompts and grading configuration. It references a material and can optionally reference a module. It also stores cognitive level, ideal answer, keywords, weight, and rounding rules, plus timestamps for creation and updates.

rubrics:
The rubrics table stores detailed scoring criteria for a single essay question. Each rubric defines aspects, descriptors, maximum score, and weights, enabling structured scoring for AI or teachers.

essay_submissions:
The essay_submissions table stores student answers to essay questions. It links each submission to a question and a student, and tracks AI grading status, errors, submission type, and attempt count.

ai_results:
The ai_results table stores AI-generated scores and feedback for a submission. It includes the raw model response and RAG logs for transparency and auditing.

teacher_reviews:
The teacher_reviews table stores human grading results that confirm or override AI scoring. Each review is tied to a submission and a teacher user, and includes revised score and feedback.

ai_api_usage_logs:
The ai_api_usage_logs table records AI API calls for monitoring usage, performance, and cost. It stores model name, status, token counts, and response time.

ai_grading_cache:
The ai_grading_cache table stores cached AI grading outputs keyed by a request hash. It improves performance and cost by reusing results for identical inputs, and tracks cache usage metrics.


Relation Descriptions (Why the links exist)

modules.material_id -> materials.id:
Each module belongs to a specific material, so modules can be grouped under a single learning resource.

essay_questions.material_id -> materials.id:
Each question is tied to a material to anchor the prompt to its learning context.

essay_questions.module_id -> modules.id:
Optional link when a question is specifically part of a module.

rubrics.question_id -> essay_questions.id:
Each rubric defines scoring for one question, ensuring criteria match the prompt.

essay_submissions.soal_id -> essay_questions.id:
Each submission answers one specific question.

essay_submissions.siswa_id -> users.id:
Each submission is owned by a student user.

ai_results.submission_id -> essay_submissions.id:
AI results are generated for a single submission.

teacher_reviews.submission_id -> essay_submissions.id:
Teacher review is attached to the submission being evaluated.

teacher_reviews.teacher_id -> users.id:
Identifies which teacher performed the review.
