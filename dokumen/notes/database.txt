Table Purposes (General, in English)

users: Stores all application users, including students, teachers, and superadmins, plus profile and preference data.
classes: Represents classes created by teachers, including settings such as announcements and join policy.
class_members: Maps users to classes and stores membership status and approval timestamps.
materials: Holds learning materials uploaded to a class.
modules: Stores uploaded module files linked to materials.
essay_questions: Stores essay questions tied to materials (optionally to modules).
rubrics: Scoring criteria for essay questions.
essay_submissions: Student answers to essay questions, including AI grading status.
ai_results: Stores AI grading output for a submission.
teacher_reviews: Final teacher review and score for a submission.
profile_change_requests: User requests to change profile data and reviewer decisions.
class_teaching_modules: Teacher-shared modules at the class level.
question_bank_entries: Reusable questions saved to a class question bank.
ai_api_usage_logs: Logs AI API calls for monitoring and cost/usage analysis.
ai_grading_cache: Caches AI grading results to avoid repeated calls for the same request.
system_settings: Key-value settings for system-wide behavior.
admin_audit_logs: Tracks admin actions for auditability.
announcements: Global announcements shown to users by role and time window.
sections: Organizes content in a class (like course sections).
section_contents: Content items inside a section (materials, questions, tasks, assessment).
grade_appeals: Student appeals for grading decisions and their resolution status.
notifications: In-app notification records per user.
rubric_templates: Reusable rubric templates authored by users.

Relation Purposes (Why the links exist)

classes.teacher_id -> users.id: Each class is owned by a teacher.
class_members.class_id -> classes.id: Each membership belongs to a specific class.
class_members.user_id -> users.id: Each membership references the user who joins the class.
materials.class_id -> classes.id: Materials are scoped to a class.
materials.uploader_id -> users.id: The user who uploaded the material.
modules.material_id -> materials.id: Modules are attached to a material.
essay_questions.material_id -> materials.id: Questions are created from a material.
essay_questions.module_id -> modules.id: Optional link to a module if the question belongs to a module.
rubrics.question_id -> essay_questions.id: Rubrics define scoring for a specific question.
essay_submissions.soal_id -> essay_questions.id: A submission answers a specific question.
essay_submissions.siswa_id -> users.id: The student who submitted the answer.
ai_results.submission_id -> essay_submissions.id: AI results belong to a submission.
teacher_reviews.submission_id -> essay_submissions.id: Teacher reviews belong to a submission.
teacher_reviews.teacher_id -> users.id: The teacher who reviewed the submission.
profile_change_requests.user_id -> users.id: The user requesting profile changes.
profile_change_requests.reviewer_id -> users.id: The reviewer (admin/teacher) who approved or rejected.
class_teaching_modules.class_id -> classes.id: Modules shared for a specific class.
class_teaching_modules.uploaded_by -> users.id: The user who uploaded the class module.
question_bank_entries.created_by -> users.id: The user who created the bank entry.
question_bank_entries.class_id -> classes.id: The class where the question bank entry belongs.
question_bank_entries.source_material_id -> materials.id: Optional link to original material.
question_bank_entries.source_question_id -> essay_questions.id: Optional link to original question.
admin_audit_logs.actor_id -> users.id: The admin/actor who performed an action.
announcements.created_by -> users.id: The user who created the announcement.
sections.class_id -> classes.id: Sections belong to a class.
section_contents.section_id -> sections.id: Content belongs to a section.
section_contents.linked_material_id -> materials.id: Optional link to a material.
grade_appeals.submission_id -> essay_submissions.id: Appeal is tied to a submission.
grade_appeals.question_id -> essay_questions.id: Appeal references the questioned item.
grade_appeals.class_id -> classes.id: Appeal is tied to the class context.
grade_appeals.student_id -> users.id: The student who submitted the appeal.
grade_appeals.resolved_by -> users.id: The teacher/admin who resolved the appeal.
notifications.user_id -> users.id: Notifications are per user.
rubric_templates.created_by -> users.id: The author of the rubric template.
